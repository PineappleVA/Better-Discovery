Acerca de Better Discovery

Better Discovery es una galería abierta y comunitaria para creaciones HTML. Sube tus páginas HTML, demos interactivas, juegos o herramientas y deja que cualquier persona las vea, juegue, reaccione y comente — sin necesidad de programar.

✨ Funciones

📤

Subir HTML

Pega hasta 1 MB de código HTML y publícalo al instante.

🖥️

Vista previa en vivo

Cada creación corre en un iframe con sandbox — incluyendo scripts.

❤️

Me gusta

Muestra apoyo a tus creaciones favoritas con un solo clic.

💬

Comentarios

Deja feedback, preguntas o apoyo a las personas creadoras.

🔍

Búsqueda

Encuentra creaciones por título o descripción con búsqueda instantánea.

📱

Responsive

Funciona en escritorio, tablet y dispositivos móviles.

⚙️ Guía de configuración (Supabase)

Better Discovery es un sitio estático que usa Supabase como backend. Necesitas un proyecto gratuito de Supabase para que funcione por completo.

1

Crea un proyecto gratuito de Supabase

Ve a supabase.com, regístrate (es gratis) y crea un proyecto nuevo. Espera a que termine de aprovisionarse.

2

Ejecuta el esquema SQL

En el panel de Supabase, ve a SQL Editor → New query, pega el bloque de abajo y haz clic en Run.

-- Better Discovery — database schema CREATE TABLE IF NOT EXISTS snippets ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, title TEXT NOT NULL, description TEXT NOT NULL DEFAULT '', html_content TEXT NOT NULL, author TEXT NOT NULL DEFAULT 'Anonymous', tag TEXT, likes INTEGER NOT NULL DEFAULT 0, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); CREATE TABLE IF NOT EXISTS comments ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, snippet_id UUID NOT NULL REFERENCES snippets(id) ON DELETE CASCADE, author TEXT NOT NULL DEFAULT 'Anonymous', content TEXT NOT NULL, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ); -- Enable Row-Level Security ALTER TABLE snippets ENABLE ROW LEVEL SECURITY; ALTER TABLE comments ENABLE ROW LEVEL SECURITY; -- Snippets: anyone can read and insert; anyone can update likes CREATE POLICY "Public read snippets" ON snippets FOR SELECT USING (true); CREATE POLICY "Public insert snippets" ON snippets FOR INSERT WITH CHECK (true); CREATE POLICY "Public update likes" ON snippets FOR UPDATE USING (true) WITH CHECK (true); -- Comments: anyone can read and insert CREATE POLICY "Public read comments" ON comments FOR SELECT USING (true); CREATE POLICY "Public insert comments" ON comments FOR INSERT WITH CHECK (true);
3

Copia tus credenciales de API

En el panel de Supabase ve a Project Settings → API. Copia la Project URL y la clave anon / pública.

4

Añade las credenciales en js/config.js

Abre el archivo y reemplaza los valores de ejemplo:

const SUPABASE_URL = 'https://abcdefghij.supabase.co'; const SUPABASE_ANON_KEY = 'eyJhbGc...(your anon key)...';
5

Despliega en GitHub Pages

Sube el repositorio a GitHub y luego ve a Settings → Pages, define como origen la rama main y la carpeta raíz, y guarda. Tu sitio quedará en https://<user>.github.io/<repo>.

📄 Páginas

El proyecto contiene cuatro páginas:

  • index.html — Explora y descubre todas las creaciones HTML
  • upload.html — Sube una nueva creación HTML (hasta 1 MB)
  • view.html?id=… — Visualiza, da like y comenta una creación
  • about.html — Esta página: resumen de funciones y configuración

🛠️ Tech Stack

HTML + CSS + JavaScript puro (sin build tools ni frameworks) y Supabase para la base de datos. Hospedado gratis en GitHub Pages.

🔒 Seguridad

Todo el HTML subido se ejecuta dentro de un <iframe> con sandbox usando un atributo sandbox restringido. Los scripts se ejecutan en un origen nulo único — no pueden acceder a cookies, localStorage ni DOM de la página padre.
