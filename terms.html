Términos y condiciones

Última actualización: 19 de junio de 2026.

Estos términos regulan el uso de Better Discovery, una galería comunitaria alojada en GitHub Pages. Al usar la plataforma aceptas estas condiciones y confirmas que entiendes cómo funciona el servicio.

1. Qué ofrece el servicio

Better Discovery permite publicar, descubrir y comentar creaciones HTML. El sitio está diseñado para funcionar como una experiencia estática en GitHub Pages, con autenticación y persistencia apoyadas en servicios externos.

2. Cuenta y responsabilidad

Eres responsable de la información que compartes, del contenido que publicas y de cualquier actividad realizada desde tu cuenta. Si detectas un uso no autorizado, debes cambiar tus credenciales y revisar tu sesión.

3. Contenido permitido

  • Creaciones HTML completas que puedan renderizarse en un navegador.
  • CSS y JavaScript embebidos dentro del propio documento HTML.
  • Contenido original o contenido sobre el que tengas derechos suficientes para publicar.

4. Contenido prohibido

  • Archivos ZIP, paquetes comprimidos u otros formatos no HTML.
  • Copias, clones o remixes no autorizados de creaciones ajenas.
  • Contenido ilegal, fraudulento, difamatorio, malicioso o que abuse de terceros.
  • Código destinado a robar credenciales, explotar la plataforma o dañar el navegador de otras personas.

5. Moderación

Podemos ocultar, rechazar o eliminar contenido que incumpla estas reglas. También podemos limitar funciones si se detecta abuso, spam, automatización indebida o comportamiento que degrade la experiencia del resto.

6. Propiedad intelectual

Conservas la propiedad de tus creaciones, pero nos concedes una licencia no exclusiva para mostrarlas, almacenarlas y reproducirlas dentro de Better Discovery con el único fin de operar el servicio.

7. Limitación de responsabilidad

El servicio se ofrece “tal cual”. No garantizamos disponibilidad continua, ausencia de errores ni que el contenido publicado por otras personas sea exacto, seguro o adecuado para todos los públicos.

8. Cambios

Podemos actualizar estas condiciones cuando sea necesario. Si el cambio es material, intentaremos reflejarlo en la interfaz o en la documentación del proyecto.
